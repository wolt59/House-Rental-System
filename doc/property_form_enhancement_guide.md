# 房源表单增强说明

## 概述
本次更新对房源发布表单进行了全面优化，将字段分为**必填信息**和**选填信息**两大类，并美化了表单UI。

---

## 一、数据库迁移

### 执行步骤
1. 登录MySQL数据库
2. 执行迁移脚本：
```bash
mysql -u root -p your_database < doc/migration_property_form_enhancement.sql
```

### 新增字段清单

#### 第一类：必填信息（业务核心数据）

**1. 位置地址类**
- `city` - 所属城市
- `community_name` - 小区名称

**2. 房屋物理属性类**
- `rental_type` - 房源类型（整租/合租/公寓/商铺）
- `bedrooms` - 卧室数量
- `livingrooms` - 客厅数量
- `bathrooms` - 卫生间数量
- `building_area` - 建筑面积（㎡）
- `usable_area` - 实用面积（㎡）

**3. 租赁价格与押金类**
- `payment_method` - 付款方式（押一付一/押一付三/半年付/年付）

**4. 租期与入住规则类**
- `min_lease_term` - 最短租期（月）
- `earliest_move_in_date` - 最早可入住时间

**5. 费用权责划分**
- `property_fee_bearer` - 物业费承担方（房东/租客）
- `utility_fee_bearer` - 水电燃气费承担方
- `other_fee_bearer` - 其他费用承担方

**6. 房屋使用限制**
- `allow_pets` - 是否允许饲养宠物（0=否，1=是）

---

#### 第二类：选填信息（丰富房源介绍）

**1. 房屋补充属性**
- `build_year` - 建筑建成年代
- `has_elevator` - 电梯配置（0=无，1=有）
- `total_households` - 总户数
- `property_management_type` - 物业类型

**2. 室内配套设施**
- `facilities` - 已存在，保持不变

**3. 周边配套描述**
- `surrounding` - 已存在，保持不变

**4. 看房时间规则**
- `viewing_time_rules` - 日常可接待看房的时间段

**5. 图文与描述内容**
- `description` - 已存在，保持不变

**6. 特色标签**
- `tags` - 特色标签（近地铁/拎包入住/采光好等，逗号分隔）

**7. 个人补充备注**
- `landlord_notes` - 房东额外说明、特殊要求等

---

## 二、前端表单优化

### 表单分区设计

#### 第一类：必填信息区域
- 📍 **位置地址**：城市、行政区、小区名称、详细门牌号
- 🏠 **房屋物理属性**：房源类型、户型、面积、楼层、朝向、装修
- 💰 **租赁价格与押金**：租金、押金、付款方式
- 📅 **租期与入住规则**：最短租期、最早入住时间
- 💳 **费用权责划分**：物业费、水电燃气费、其他费用承担方
- ⚠️ **房屋使用限制**：是否允许宠物
- 📝 **基础展示**：房源标题、详细描述

#### 第二类：选填信息区域
- 🏢 **房屋补充属性**：建筑年代、电梯、总户数、物业类型
- 🛋️ **室内配套设施**：设施清单
- 🌳 **周边配套**：周边环境描述
- 🕐 **看房时间**：可看房时间段
- 🏷️ **特色标签**：自定义标签
- 💬 **房东备注**：补充说明

### UI美化特性

1. **分区清晰**：使用卡片式布局，每个类别独立成区
2. **视觉层次**：必填区域使用浅灰背景，选填区域使用虚线边框
3. **图标引导**：每个子分类配有emoji图标，提升可读性
4. **滚动优化**：表单高度限制为70vh，支持内部滚动
5. **输入组件优化**：
   - 数字字段使用 `el-input-number` 组件
   - 选项字段使用 `el-select` 下拉选择
   - 日期字段使用 `el-date-picker`
   - 布尔值使用 `el-radio-group`

## 三、房源详情页优化

### 新增展示内容

1. **特色标签**：在标题下方以tag形式展示
2. **详细信息分组**：
   - 位置信息：城市、区域、小区、详细地址
   - 房屋属性：类型、户型、面积、朝向、楼层、装修
   - 租赁信息：租金、押金、付款方式、租期
   - 费用划分：物业费、水电燃气、其他费用
   - 使用限制：是否允许宠物

3. **折叠面板展示选填信息**：
   - 🏢 房屋补充信息（建筑年代、电梯、户数、物业）
   - 🛋️ 室内设施
   - 🌳 周边配套
   - 🕐 看房时间

4. **房东备注**：以灰色背景块单独展示

### 智能格式化
- **户型显示**：自动组合卧室/客厅/卫生间数量（如：2室1厅1卫）
- **楼层显示**：自动组合格式为"所在楼层/总楼层"（如：5/18）
- **面积显示**：优先显示建筑面积，其次显示兼容的area字段

## 五、核心字段校验

修改以下核心字段时，如果房源已审核通过，会自动转为"待审核"状态：
- address（详细地址）
- city（城市）
- region（行政区）
- community_name（小区名称）
- building_area（建筑面积）
- usable_area（实用面积）
- bedrooms（卧室数）
- livingrooms（客厅数）
- bathrooms（卫生间数）
- rent（租金）
- deposit（押金）
- floor_number（所在楼层）
- total_floors（总楼层）

## 六、使用建议

### 房东端
1. **必填信息务必准确填写**：这些信息会同步到电子合同，影响租客筛选和定价
2. **选填信息尽量完善**：可以提升房源吸引力，增加曝光率
3. **特色标签要精准**：有助于租客快速找到符合需求的房源
4. **看房时间要明确**：方便租客预约，提高看房效率

### 租客端
1. **关注特色标签**：快速筛选符合需求的房源
2. **查看费用划分**：明确各项费用承担方，避免后续纠纷
3. **阅读房东备注**：了解特殊要求和说明

---

## 七、注意事项

1. **数据迁移后**：现有房源的新增字段均为NULL，需要房东逐步补充
2. **兼容性**：保留了旧的`area`和`property_type`字段，确保旧数据正常显示
3. **合同同步**：第一类必填信息会自动同步到电子合同，请确保准确性
4. **审核机制**：修改核心字段会触发重新审核，请提前告知房东
