# 房源表单增强 - 快速执行指南

## 🚀 快速开始

### 步骤1：执行数据库迁移

```bash
# 进入项目目录
cd D:\CODE\House-Rental-System

# 执行数据库迁移（根据你的数据库配置修改用户名和密码）
mysql -u root -p house_rental < doc/migration_property_form_enhancement.sql
```

或者直接在MySQL客户端中执行：
```sql
source D:/CODE/House-Rental-System/doc/migration_property_form_enhancement.sql;
```

### 步骤2：重启后端服务

```bash
# 如果使用PowerShell
.\start_backend.ps1

# 如果使用普通命令行
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 步骤3：重启前端服务

```bash
# 进入前端目录
cd frontend

# 如果前端已在运行，停止后重新启动
npm run dev
```

## 🔧 常见问题

### Q1: 数据库迁移失败
**错误**：`Duplicate column name 'city'`
**解决**：说明字段已存在，可以跳过此步骤或手动检查表结构

### Q2: 前端表单不显示新字段
**解决**：
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 硬刷新页面（Ctrl+F5）
3. 检查前端控制台是否有报错

### Q3: 提交表单时出现422错误
**解决**：
1. 检查必填字段是否都已填写
2. 确认数字字段格式正确
3. 查看浏览器Network面板的请求详情

### Q4: 旧房源数据显示异常
**说明**：这是正常的，旧房源的新增字段为NULL
**建议**：提示房东编辑房源时补充完整信息
